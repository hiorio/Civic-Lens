# Bill Status Dictionary

의안 처리상태는 원천 상태값과 내부 enum, 사용자 표시 문구를 분리한다. 원천값은 계속 추가될 수 있으므로 매핑되지 않는 값은 `UNKNOWN`으로 저장하고 원천 문자열은 `BillStatusHistory.rawStatus`에 보존한다.

| 원천 상태 예시 | 내부 상태 enum | 사용자 표시 문구 | 쉬운 설명 |
| --- | --- | --- | --- |
| 접수 | `RECEIVED` | 접수 | 의안이 국회에 접수되었습니다. |
| 위원회 회부 | `REFERRED_TO_COMMITTEE` | 위원회 회부 | 관련 위원회로 심사가 넘어갔습니다. |
| 위원회 심사 | `UNDER_COMMITTEE_REVIEW` | 위원회 심사 중 | 위원회에서 내용을 검토하고 있습니다. |
| 위원회 통과 | `PASSED_COMMITTEE` | 위원회 통과 | 위원회 심사를 통과했습니다. |
| 본회의 통과, 가결 | `PASSED_PLENARY` | 본회의 통과 | 본회의에서 통과 또는 가결되었습니다. |
| 공포 | `PROMULGATED` | 공포 | 법률로 공포되었습니다. |
| 폐기, 임기만료폐기 | `DISCARDED` | 폐기 | 더 이상 처리되지 않는 상태입니다. |
| 철회 | `WITHDRAWN` | 철회 | 발의자가 의안을 거두었습니다. |
| 빈 값 또는 미확인 값 | `UNKNOWN` | 확인 필요 | 원천 상태를 아직 내부 상태로 매핑하지 못했습니다. |

## PoC 확인 결과

- 최근 의안 100건에서는 처리상태가 비어 있는 경우가 많아 `UNKNOWN` 저장이 필요하다.
- 원천 상태값 샘플이 더 쌓이면 이 문서를 먼저 갱신한 뒤 `mapBillStatus` 매핑을 확장한다.
